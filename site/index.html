Writing System Explorer

A read-only projection of the Context Engineering Kit. Human-legible. AI-debuggable. Version-aware.

This site exists to make the system visible, navigable, and auditable — without changing how or where truth is authored.

What This Is

This site is a system explorer for a structured writing framework designed to prevent:

  • Generic “AI voice” in generated content
  • Invented claims or hallucinated offers
  • Inconsistent messaging across assets
  • Format drift across content types

Every page on this site renders a file directly from the repository, preserving structure, version information, and trust levels.

What This Is Not

This is not:

  • A CMS or content management system
  • A marketing site or public blog
  • A documentation generator that owns truth
  • An authoring or editing environment

Files in the repository remain canonical. This site is a view layer only.

Navigation

System Overview

Explains the architecture, design principles, and workflow. Start here to understand how the system works end to end.

Context Profiles

Core JSON files that define voice, audience, offers, and claims constraints. These are first-class sources of truth.

Prompts

Structured interview prompts used to generate context profiles. These define how truth enters the system.

Skills

Content-type expertise packages (landing pages, emails, case studies). Skills define format and strategy — not voice or claims.

Knowledge

Reference content, templates, and examples. Organized by trust level (gold-standard, templates, drafts, notes).

Design Principles

  • Structure over polish — Correctness beats aesthetics
  • Navigation mirrors roles — The site reflects how the system is used
  • Files remain canonical — The repository is the source of truth
  • Read-only by design — This frontend never mutates data
  • Clarity over cleverness — Precision beats persuasion
